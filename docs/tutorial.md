# UltraRAG4 独立教程页内容稿

本文件对应独立页面 `/enterprise/tutorial` 的内容结构。页面本身不依赖企业端侧边栏，不引导跳转到模型、工具、聊天端等其他页面；所有主体内容都在单页内通过锚点阅读。

## 定位

UltraRAG4 是面向业务流程的企业数字员工运行时。

它的核心不是“能聊天”，而是让一个真实岗位拥有自己的配置、资源、执行规则和运营记录。每个员工都可以有独立知识、SOP、工具和记忆；每次对话都能回看路由、工具调用、回复和反馈。

## 核心功能

### 数字员工

每个员工独立维护岗位描述、服务范围、资源绑定和运营记录，适合把客服、导购、运营、知识助手拆成不同角色。

### 知识库

把制度、商品、交付和服务口径解析为可检索片段，回复时保留引用线索，降低“听起来对但查不到来源”的风险。

### 技能

把浏览器、查询、文档处理、MCP 或项目工作流沉淀为技能，让能力能被多个员工复用和迭代。

### SOP

用节点、必填信息、允许动作、中断策略和回复规则描述流程；已满足的信息不重复追问，只推进真正缺失的部分。

### 工具

通过 HTTP 工具和内置工具连接订单、商品、知识检索或内部服务，让员工可以查询、校验、创建和触发动作。

### 记忆

把用户偏好、项目背景、复盘结论和协作习惯沉淀为可查记录，帮助后续会话继承已验证的上下文。

### 定时任务

让员工按一次性、每日、每周或每月计划执行提示词，适合巡检、周报、提醒、异常跟进和周期分析。

### 追踪与反馈

用对话日志、Trace、反馈分析和事件记录串起路由、工具调用、回复与改进线索，失败后能定位到具体环节。

## 快速开始

1. 建立运行底座：准备 OpenAI 兼容模型、单端口服务和 demo 租户数据，让企业端、聊天端、API 文档在同一个本地入口下运行。
2. 定义一个真实岗位：先选一个低风险但流程密集的岗位，例如售后服务、导购、运营巡检或内部知识助手。
3. 补齐知识与 SOP：把业务文档沉淀为知识库，把关键流程拆成节点、必填信息、允许动作和回复规则。
4. 连接必要工具：只给员工接入它真正需要的工具，例如订单查询、商品购买、内部查询或知识检索。
5. 用真实表达试跑：覆盖缺信息、信息已给全、改口、插话和转人工。
6. 把结果变成运营资产：通过 Trace、反馈、记忆和定时任务沉淀失败原因、稳定口径和长期任务。

## 架构

```text
入口层 -> 配置层 -> 运行层 -> 上下文层 -> 观测层
```

执行流程：

```text
User message
  -> Chat API
  -> Router
  -> Skill / SOP / Knowledge / Tool
  -> Agent Loop
  -> Response Generator
  -> Trace / Feedback / Memory
```

## 案例

- 售后服务员工：售后 SOP -> 订单查询工具 -> 风险回复规则 -> Trace 复盘
- 电商导购员工：商品知识库 -> 比价技能 -> 购买 SOP -> 工具结果反馈
- 运营常驻员工：定时任务 -> 运行记录 -> 记忆沉淀 -> 下轮复用
- 内部知识助手：知识库 -> 引用回复 -> 反馈分析 -> 知识更新
